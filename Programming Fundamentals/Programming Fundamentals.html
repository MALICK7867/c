A problem is a situation or matter that requires proper attention to be solved. It is a process of identifying a problem and finding its best solution. It is a skill that can be developed by following a well-organized approach.

What is Program

A program is a set of instructions given to a computer to perform a specific task. For example, sorting numbers, calculating grades are basic examples of a program. A process of writing code in order to give instructions to a computer is called Programming.

Advantages of Computer Program

Some advantages of a computer program are as following:

  • Computer program can solve many problems by giving instructions to the computer.
  • It can be used to perform a task repeatedly and quickly.
  • It can process large amount of data easily.
  • It can display the result in different styles.
  • The processing of program is more efficient andless time consuming.
  • Different types of programs are used in different fields to perform certain tasks.

Programming Language

A programming language is a notational system for describing instructions for computer in a machine-readable and human-readable form. There are many programming languages such as C, C++, Java, Python, Assembly etc. However, each programming language has its own programming rules, sytax and structures.

Levels of Programming Languages

Low Level Language

The programming languages which are easily understandable to machines but difficult to understand for human. There are two types of low level programming. ie.

  • Machine Language
  • Assembly Language

Machine Language

Machine Language consist of binary numbers (1,0). It is difficult for a human being to write codes in machine language. Program writing, reading, debugging and upgrading in machine language is very difficult for human. Therefore the programming in Machine Language is too slow.

Assembly Language

Assembly language uses symbolic codes which easy to remember such as ADD, SUM, MOV etc. Theses symbolic codes are developed against some binary codes. So writing, reading, debugging and upgrading the programs in Assembly language become easy as compare to the Binary Language.

High Level Language

To speed up the programming process High level Languages are developed. These languages are human friendly because these are more close to the human language. High-level languages, such as C, C++, Java, C#, Swift and Visual Basic, allow to write instructions that more like English and mathematical notations. However, High Level Languages are NOT directly understandable by the machines. They need to be translated into machine level. These languages are machine independent; i-e portable.

Difference Between Low and High level programming

The difference between low level and high level programming language is as following:
High Level Language Low Level Language
1. High-level Languages are very easy to learn. 1. Low-level languages are difficult to learn.
2. High-level Languages are near to human languages. 2. Low-level languages are far from the human languages
3. Programs in high-level Languages are slow in execution. 3. Programs in low-level languages are fast in execution.
4. Programs in high-level Languages are easy to modify. 4. Programs in low-level languages are difficult to modify
5. High-level Languages do not provide much facility at the hardware level. 5. Low-level languages provides facility to write programs at hardware level.
6. Deep knowledge of hardware is not required to write programs. 6. Deep knowledge of hardware is required to write programs.
7. These languages are normally used to write application programs. 7. These languages normally used to write hardware programs.

Types of Code

There are two types of codes. ie.

Source Code

A program written in a high-level languages are also known as source code. Source code is also known as source program. Computer cannot understand the statements of high-level language. The source code can not be excuted by the computer directly. It must be converted into object code and then executed.

Object Code

A program in machine language is called object code. It is also called object program or machine code. Computer understand the object code directly.

Language Processor

A language processor is a type of software program that translates the High Level Language program (source code) into Machine Language (object code). Language processor is also called language translator.

  • Source Code: The program written by the user.
  • Object Code: The program translated into machine language.

Compiler

Compiler converts program written in a high-level programming language into the machine code, at once, before program runs.

Interpreter

Interpreter converts each high-level program statement, one by one, into the machine code, during program run. Scripting languages JavaScript and PHP are processed by Interpreters.

Assembler

A type of converter that converts Assembly Language program into machine code.

History of C++

Background of C++

Many new programming languages appeared during the 1960s. The computers at that time were still in early stage of development. The language ALGOL60 was developed as an alternative to FORTAN.
Ken Thompson createdB language in 1970. C language was derived from B language. The B language provided the basis for the development of C. The C language was originally designed to write programs under UNIX operating system.

Bjarne Stroustrup from Bell Labs started the development of C++ language in 1980. The first commercial release of C++ appeared in October 1985. It was originally named "C with classes". It was an improved version of C which allows the use of programming technique known as object oriented programming (OOP).

Features of C++

Some important features of C++ are mentioned:

Convinient Language

C++ is very convinient language. It provides many facilities in easier way that are diffiluct to use in low level languages. The programmers can write more complex codes more easily as compared to low-level languages.

Well Structured Language

C++ is well structured language. Its syntax is very easy to understand. The programs written in C++ language are very easy to maintain and modify.

Case Sensitivity

C++ is case sensitive language. It means that it can differentiate uppercase and lowercase words. All keywords are written in lowercase. This feature makes it easier to maintain the code.

Machine Independence

C++ language provides machine independence. It means that the programs written in C++ language can be executed on different types of computers.

Object Oriented

C++ is an object-oriented language. In object-oriented technique, the programs are written on the basis of objects. An object is a collection of data and function.

Modular Programming

C++ language provides a facility of modular programming which means that the program can be divided into modules which increases the maintainablity of code. These modules can be developed and compiled independently and then linked together.

Standard Libraries

The standard C++ library is a set of functions, constants, classes and objects to extend C++ language.

Hardware Control

C++ provides cloase control on hardware. The programmer can write efficient programs to control hardware components of computer system.

Brevity

C++ is a small language. It has a small number of keywords and programming controls but still very powerful for developing different types of programs. The code written in C++ is very small as compared to other languages.

Speed

C++ compilers generate very fast code. The resulting code from a C++ compilation is very efficient. The program executes very efficiently and program takes less time to execute.

Popular Language

C++ is a very widely used programming language. There are many tools available for C++ programming and a broad base of programmers contributing to C++ community.

C Compatibility

C++ is backward compatible with C language. Any code written in C language can easily be included in a C++ program without hardly making any changes.

Basic Structure of C++ Program

The format of writing programs in C++ is called structure. The basic structure of C++ programs in very flexible. A program in C++ consists of following parts:

  • Preprocessor directive
  • Main() function
  • Program body (C++ statements)

Preprocessor Directive

Preprocessive directives are instructions given to the preprocessor. The preprocessor directives are processed by a program known as preprocessor. It is part of C++ compiler. It is part of C++ compiler. It modifies C++ source program before compilation. The semicolon is not used at the end of preprocessor directives. The preprocessor directives start with hash symbol (#). These directives are written at the start of program.

Include Preprocessor

Include preprocessor directive is used to include header files in the program. The syntax of using this directive is as follows:

#include<iostream>

The above statement tells the compiler to include the file iostream in the source program before compiling it.

Header Files

Header files are collection of standard library functions and objects to perform different tasks. C++ provides many header files for different purposes. Each header file contains different type of predefined functions and objects. Many header files can be included in one program. The header file must be included in the program before calling any of its functions. The name of header file is written in angle brackets. These files are provided by C++ compiler system.
Syntax:
The syntax of header file is as following:

#include <header file>
The name of header file can also be used in double quotes as follows:
#include "header file"

Example 1:

#include <iostream>
The above statement will include the header file named iostream in the program. The word iostream means input/output stream.

Main() Function

The main() function is the starting point of a C++ program. The control enters main() function and starts executing its statements when the program is run. A program that do not have main() function can be compiled but can not be executed. The syntax of main function is:

int main()
{
    body of main function
}

Datatypes

Variables

During programming we need to store data like marks, names, etc. To store these values we use variables. These variables are located in the memory of system. This memory is divided into blocks and each block has its own unique numberical address. To store and access these blocks of the memory, we give them a name and these are called variables. These are called variables because the value is different or may change and store different values at different time. These variables are declared using these rules:

  • Type of variable
  • A name of variable
  • Size of variable
  • Value stored in variable

Variable Type

A variable must have a datatype associated with it. It is used to identify which type of data is stored in a variable. For example:

  • Integer (used for storing whole numbers)
  • Float (used for storing numbers, decimal points)
  • Character (used to store alphabets)
  • Boolean (true/false)
  • String (used to store text)

Variable Size

Different variables have different size. It is a datatype which tells us about the size of the variable. It depends on the IDE to assign size to a specific datatype. Here, we will discuss the size of different variables used in Dev C++. ie.

  • Integer (int) = 4 Bytes
  • Float (float) = 4 Bytes
  • Double (double) = 8 Bytes
  • Character (char) = 1 Byte
  • String (string) = 8 Bytes
  • Boolean (bool) = 1 Byte

Variable Name

We use naming conventions to declare variable names. Variable should be given name which matches the nature of the data. Variable names should be legal. For example:

  • Names can contain series oof characters consisting of letters, digits and underscores(_)
  • It can not be started by a number or digit or any special character.
  • Spaces are not allowed
  • Keywords or reserved words are not allowed
  • Uppercase and lowercase letters are two different characters. So, a1 and A1 are different.

Scope of Variables

In terms of scope, variables are categorized into two broad types:

Local Variables

Local variables are those whcih are declared in the body of a function. These variables are accessible within the body of the function.

Global Variables

These variables are declared outside the body of all functions. These variables are accessible anywhere in the program.

Variables and Constants

Those quantities whose values can be changed throughout the program are known as Variables.

Those quantities whose values CANNOT be changed in the program.

Types of Errors

What is an Error

Programming errors are referred to as mistakes or bugs in the program. There are three types of errors:

  1. Syntax Error
  2. Runtime Error
  3. Logical Error

Syntax Error

A type of error that occurs at the time of compilation. It usually occurs due to an error in the rules & regulation or structure of the program. It do not allow the program to get compiled and as a result not result in execution of program. It is very easy to find because compiler programs usually highlight the error.

Runtime Error

A type of error that occur at the time of execution. Occurs due to an illegal operation in the program such as dividing by zero, referencing an invalid file, calling invalid function. It cause the program to stop or crash. It is diffult to identify as compared to Syntax error.

Logical Error

A type of error that occur at the time of execution. It usually occur due to incorrect logic or fault in the algorithm in the program such as incorrect sequence of instuctions. It do not stop the compilation or execution of the program but gives wrong output of the program. It is very hard and complex to identify.

Life-Cycle of Program

Introduction

Program lifecycle consist of the stages in which a program undergoes from initial creation to deployment and execution. These phases consist of:

  • Program Editting
  • Pre-Processing
  • Compilation
  • Linking
  • Loading
  • Execution

Full Lifecycle

  • Program Editing: Program is created in the editor and stored on disk as .cpp file
  • Preprocessing: Preprocessor program processes the code
  • Compilation: Compiler creates object code and stores it on disk as .obj file
  • Linking: Linker links the object code with the libraries and stores as .exe file
  • Loading: Loader puts program in memory
  • Execution: CPU takes each instruction and executes it, possibly storing new data values as the program executes

Type Conversion

Implicit Type Conversion

Type conversion allows a compiler to convert one datatype at the time of compilation of a program. This is called implicit type conversion.
There is a data loss when using this type of conversion. Data loss happens when data of larger type is converted into smaller type.

Rules

There are some rules which must be kept in mind while using type conversion. These are:
When operating with values of different data types, the lower ranked value will be promoted to higher data type.
When using assignment operator (=), the type of expression on right side will be converted into type of left side variable.

Truncate

When a higher or upper datatype is converted into lower or smaller datatype, it gets demoted and may lose data. This process is called Truncate and datatype is truncated. For example: when an floating point value is converted into integer type value, it gets truncated.

Promote

When a smaller or lower datatype is converted into higher or upper datatype, it gets Promoted. This makes values more accurate. For example: when an integer is converted to floating point value, is gets promoted.

Explicit Type Conversion

When the user manually changes data from one type to another, this is known as type casting. This type of conversion is also called explicit type conversion. Syntax:

Static_cast<DataType>(Expression);
OR
(DataType) Expression;
