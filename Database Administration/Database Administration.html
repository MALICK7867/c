Introduction

What is Database

A collection of information organized in such a way that a computer program can quickly select desired piece of data. Database is just like electronical way of storing files. A database is a structured collection of data that is organized and stored in a way that allows for efficient retrieval, management, and manipulation of that data. Databases play a crucial role in modern information systems, serving as the backbone for data storage and retrieval in applications, websites, and virtually any system that deals with structured data.

Database Management System

A software used to manage or use database in a computer system is known as DBMS or database management system. It allows you to create and manage database same like Microsoft Word helps you to create and manage word files or documents. It is a computer software which provide interface between a user and database. It is a software which is used to create, update, query and do administration process in database. Some examples of DBMS are: Oracle, MySQL, PostgreSQL, Ms SQL server, Firebird, etc.

Terms in DBMS

Entities

An entity is a distinct and uniquely identifiable object or concept, typically representing a real-world item or an abstract idea. In databases and data modeling, entities have attributes that describe them, and they can be related to other entities to represent how they interact. Some examples of entities are: Employee, student, places, etc.

Attributes

Attributes are characteristics or properties of an entity that provide information about that entity. Attributes describe specific details or features of an entity, such as its name, age, or address. For example: Employee_name, Employee_salary, etc.

Relationship

Relationship is an association between two or more entities and describe how these entities are related to each other. There are some common types of relationships. ie.

  • One to One
  • One to Many
  • Many to Many

Keys in Database

Primary Key

A primary key is a unique identifier for each row (record) in a table. It ensures that each row is uniquely identifiable within the table. A primary key can consist of one or multiple columns. Primary keys are used to make sure that no data is repeated or replicated.

Super Key

A super key is a set of one or more columns that can be used to uniquely identify records in a table. It may contain more columns than necessary to uniquely identify records.

Difference Between Super and Primary Key

Primary Key:
  • A primary key is a specific type of key used in a relational database.
  • It is a minimal set of one or more columns that uniquely identifies each record (row) in a table.
  • A primary key enforces the uniqueness constraint, meaning that no two rows in the table can have the same values in the primary key columns.
  • The primary key is used to establish the integrity and relationships of the data within the table and can be used as a reference (via foreign keys) in other tables.
  • There can be only one primary key per table.
  • Typically, primary keys are chosen based on the business requirements and should have a real-world meaning or significance.
Super Key:
  • A super key is a broader concept and is not limited to a specific key type.
  • It is any set of one or more columns in a table that can be used to uniquely identify rows within that table.
  • A super key may or may not be minimal, meaning it may contain more columns than necessary to guarantee uniqueness.
  • Super keys are not necessarily used as primary keys.
  • They are often used for indexing, grouping, and querying purposes in the database.
  • Unlike primary keys, super keys do not impose the uniqueness constraint by themselves.

Composite Key

A composite key is a primary key that consists of two or more columns in a table. It is used when a single column cannot uniquely identify records, but the combination of multiple columns can.

Candidate Key

Candidate keys are columns or sets of columns in a table that can potentially serve as the primary key. They meet the uniqueness requirement, but one is selected as the primary key.

Foreign Key

A foreign key is a column or a set of columns in one table that refers to the primary key in another table. It establishes relationships between tables and enforces referential integrity.

Alternate Key

An alternate key is a candidate key that was not chosen as the primary key. It can still be used to uniquely identify records in the table.

Data Definition Language

Introduction

A language specifically used to define the structure or schema of a database is called data definition language. DDL, or Data Definition Language, is a subset of SQL used for defining and managing the structure of a database. DDL statements are used to create, modify, and delete database objects, such as tables, indexes, and constraints. DDL is essential for designing the database schema and ensuring data integrity. DDL statements are typically executed by database administrators and developers during the design and maintenance of a database. They are essential for defining the structure of the database and ensuring that it functions correctly and efficiently. These structures include:

  • Create
  • Drop
  • Alter
  • Truncate

Create Command

To create a database using DDL we can use the following command:

CREATE DATABASE dbname;

To use a database using DDL we can use the following command:

USE dbname;

To create a table using DDL we can use the following command:

CREATE TABLE table_name (
column_name1 datatype(size),
column_name2 datatype(size),
column_name3 datatype(size),
PRIMARY KEY (column_name1)
);

For example:

CREATE TABLE persons (
personID identity(1,1),
FirstName varchar(255),
Address varchar(255),
City varchar(255),
PRIMARY KEY (personID)
);

Alter Command

The ALTER statement allows you to modify existing database objects, such as adding or dropping columns from a table, or changing the data type of a column. To add a new column to a table we can use the following command:

ALTER TABLE persons
ADD COLUMN email VARCHAR(60);

To drop a column from a table we can use this code:

ALTER TABLE persons DROP COLUMN city;

To change name of a column from a table we can use this code:

ALTER TABLE persons CHANGE firstname FullName VARCHAR(20);

Drop Command

The DROP command is used in a database management system (DBMS) to delete database objects, such as tables, indexes, views, or even entire databases. The specific usage and behavior of the DROP command depend on what you want to delete.
To delete a table from database we use following command:

DROP TABLE table_name;

DROP DATABASE is used to delete an entire database, including all of its tables, indexes, and other objects. This is a powerful and potentially destructive command, so it should be used with caution. To delete a database we use:

DROP DATABASE database_name;

Data Manupulation Language

Introduction

DML (Data Manipulation Language) in a database is a subset of SQL (Structured Query Language) that deals with the retrieval and manipulation of data within a database. DML commands allow you to perform operations on the data stored in the database, including selecting, inserting, updating, and deleting records.

Insert Command

The INSERT statement is used to add new records (rows) into a table. You specify the values for each column or provide a subquery to insert data.

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

Update Command

The UPDATE statement is used to modify existing records in a table. You specify the columns to be updated and the new values, as well as a condition to identify the records to be updated.

UPDATE table_name
SET column1 = new_value
WHERE condition;

Delete Command

The DELETE statement is used to remove records from a table. You specify a condition to identify which records to delete.

DELETE FROM table_name
WHERE condition;

Select Command

The SELECT statement is used to retrieve data from one or more tables in the database. It allows you to specify which columns to retrieve, conditions for filtering data, and the order in which the results are presented.

SELECT column1, column2
FROM table_name
WHERE condition;

Data Control Language

Introduction

DCL (Data Control Language) is a subset of SQL (Structured Query Language) that deals with controlling and managing access to the database. DCL commands are used to set and manage permissions, privileges, and access rights for users and roles. There are two main roles in DCL. These are GRANT and REVOKE.

Grant Command

The GRANT statement is used to provide specific privileges and permissions to users or roles within the database. It allows database administrators to grant various permissions, such as SELECT, INSERT, UPDATE, DELETE, or specific object privileges, to specific users or roles.

GRANT ALL/SELECT ON table_name TO 'somename'@'somehost';

Revoke Command

The REVOKE statement is used to take away previously granted privileges from users or roles. It allows administrators to revoke previously assigned permissions.

REVOKE ALL/SELECT ON table_name FROM 'jeff'@'localhost';
